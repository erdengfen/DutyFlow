# TESTING.md

本文档记录项目测试规范、阶段验收要求和测试异常记录规则。

## 1. 基本原则

- 测试必须服务于 Demo 闭环：事件进入、身份与来源补全、权重判断、审批、任务状态、用户回馈、本地留痕。
- 测试结果必须真实记录。未彻底通过的测试不得被描述为通过。
- 测试过程中如有疑问、环境限制、外部服务不可用或其他不可抗力，必须记录在本文档的“测试问题记录”中。
- 不允许忽略失败、跳过关键链路或用未验证假设替代测试结果。

## 2. Python 文件测试入口

- 每一个 `.py` 文件都必须提供 `if __name__ == "__main__":` 入口。
- 该入口用于执行该文件内最小可运行的单元测试、示例检查或自检逻辑。
- 入口测试应聚焦本文件职责，不应依赖完整外部链路。
- 入口测试不得执行高风险外部写入、代表用户发言或绕过审批的动作。

## 3. 功能分块测试

- 每一个完整功能分块都必须在项目根目录的 `test/` 文件夹下有独立测试文件。
- 功能分块测试应覆盖该模块的正常路径、关键边界条件和失败处理。
- 涉及权限、审批、任务状态或持久化的功能，必须验证可见性、可追踪性和状态一致性。
- 测试文件命名应能直接看出被测功能，不使用含糊名称。

## 4. 阶段验收

- 每完成开发计划中的一个 step，必须执行一次完整链路检查。
- 完整链路至少验证：输入接收、身份与来源补全、权重判断、上下文处理、审批判断、任务状态更新、用户回馈、本地记录。
- 只有完整链路和该阶段相关功能测试均通过，才算该阶段验收通过。
- 如果完整链路因外部条件无法真实执行，必须记录原因、已验证部分、未验证部分和后续补测条件。

## 5. 测试记录要求

每次阶段验收或关键测试后，应记录：

- 测试日期
- 测试范围
- 执行方式
- 通过项
- 失败项
- 未验证项
- 后续处理
